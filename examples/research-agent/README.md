# Example: research-agent

Готовый research-агент, собранный по структуре статьи. Использует web_search + web_fetch, опционально подключается к internal-инструментам (Google Drive, Gmail, Slack).

## Что делает

Отвечает на исследовательские вопросы:
- «Сравни X и Y по критериям А, Б, В»
- «Что нового по теме Z за последний год»
- «Правда ли, что…»
- «Найди свежие данные по…»

Перед началом классифицирует тип запроса (depth-first / breadth-first / straightforward), потом подбирает стратегию поиска.

## Что подкручено относительно главного шаблона

- **Триггеры в `description`** перечислены явно: «find out about», «compare», «what's the latest», «is it true that», и так далее. Это поднимает шанс срабатывания.
- **Бюджет настроен под research**: до 5 / 5 / 10 / 15 вызовов в зависимости от сложности. Hard cap 20 / 100 sources — стандартный для Anthropic.
- **Source quality расписано подробно**: 6 красных флагов, явная инструкция отмечать конфликты.
- **Output format включает секцию «Conflicts and uncertainty»** — research-агенты часто находят противоречивые источники, и это надо явно отдавать пользователю, а не прятать.

## Как использовать

См. `../../docs/how-to-use.md` для подключения к Claude.ai или Claude Code.

После подключения — задавай вопросы естественным языком:
- «Сравни три облачных провайдера по ценам и фичам»
- «Что произошло с context engineering за последний год»
- «Кто сейчас CEO компании X»

Агент классифицирует запрос, выберет стратегию и вернёт структурированный отчёт.

## Что менять под себя

Если адаптируешь под свою задачу:

1. **`description`** — перепиши под триггеры твоей предметной области
2. **`<source_quality>`** — добавь красные флаги, специфичные для твоей области (финансы, медицина, юриспруденция требуют разных критериев)
3. **`<tool_selection>`** — добавь MCP-серверы или кастомные API, которые есть у тебя
4. **Бюджет и hard limit** — подстрой под свой бюджет в долларах. Для коммерческого продакшена с дорогими моделями имеет смысл начинать с более скромных лимитов
