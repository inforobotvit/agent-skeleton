# Как использовать готовый skill

Skills работают в нескольких продуктах Anthropic. Ниже — короткие инструкции для каждого.

## Claude.ai (веб + мобильное приложение)

Skills доступны на тарифах Pro, Max, Team, Enterprise.

1. Зайди в **Settings → Capabilities → Skills**
2. Нажми **Upload skill** или **Add skill**
3. Выбери папку со своим `SKILL.md` (для этого репозитория — папку `examples/research-agent/` или корневую с твоим адаптированным `SKILL.md`)
4. Skill появится в списке доступных. Дальше он подгружается автоматически по триггеру из `description`

После установки можно проверить, что skill подгрузился, спросив у Claude: «What skills do you have available?»

## Claude Code

Skills работают в Claude Code на уровне репозитория и на уровне пользователя.

**Уровень репозитория** (skill доступен только в этом проекте):
```bash
mkdir -p .claude/skills/research-agent
cp -r examples/research-agent/* .claude/skills/research-agent/
```

**Глобальный уровень** (skill доступен во всех проектах):
```bash
mkdir -p ~/.claude/skills/research-agent
cp -r examples/research-agent/* ~/.claude/skills/research-agent/
```

После этого запусти Claude Code в нужной папке, и skill будет доступен.

## Claude Agent SDK

Если ты собираешь агента через Python или TypeScript SDK, skill подключается как часть промпта при инициализации агента. См. документацию Agent SDK на `docs.claude.com`.

## MCP-сервер для skills

Если skill нужен сразу в нескольких приложениях — можно поднять его как MCP-сервер. Это уже инфраструктурное решение, выходящее за рамки этого репозитория, но как направление — упомяну.

## Проверка работы

После подключения проверь:
1. Skill отображается в списке доступных
2. При запросе, который должен триггерить skill, Claude действительно его подгружает (можно спросить «Are you using the research-agent skill right now?»)
3. Поведение агента соответствует структуре в `SKILL.md`

Если skill не подгружается — проверь `description`. Это главный триггер. Чем подробнее перечислены варианты запроса, тем выше шанс срабатывания. См. главный `README.md` в корне репозитория, секцию «Как пользоваться».

## Когда skill не работает как ожидается

Чаще всего проблемы из-за:
1. **Слишком общий `description`** — Claude не уверен, нужно ли подгружать skill. Уточни триггеры.
2. **Скилл противоречит другим инструкциям** — если у тебя в Claude.ai есть User preferences, конфликтующие с skill, выигрывают User preferences. Проверь.
3. **Слишком длинный `SKILL.md`** — теоретически Skills поддерживают любой объём, но на практике инструкции длиннее 5–8K токенов начинают «теряться». Если у тебя такой объём — раздели на основной `SKILL.md` + дополнительные файлы, которые подгружаются по необходимости (прогрессивное раскрытие).
